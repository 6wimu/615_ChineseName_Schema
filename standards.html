Relevant Metadata Standards

Standards and semantic models used directly or as design references.

OverviewImplementation noteValidation strategy

Standards overview

Dublin Core TermsUsed directly for record identifiers and descriptive naming rationales through xs:import and ref=.
Schema.orgInforms the meaning of person-name and nationality concepts. The final XSD declares corresponding project elements locally.
FOAFInforms the distinction between family name and given name in person-name modeling.
SKOSInforms definition writing and the organization of controlled vocabularies.
ISO 3166-1Provides the alpha-2 format used for nationality values such as KR, US, GB, and JP.
IPAProvides an optional pronunciation layer beyond Pinyin for precise phonetic description.

Implementation note

This schema is an application profile: it combines an imported external namespace with locally defined elements created for the Chinese-name recommendation domain. Schema.org and FOAF are RDF-oriented vocabularies and are not imported as XSD dependencies here; instead, their semantics inform local elements such as familyName and givenName.

Validation strategy

cn_schema.xsd imports the separate dcterms.xsd file and references dcterms:identifier and dcterms:description as global elements. The local dependency is deliberately limited to those two terms so GitHub Pages and VS Code can validate the project without a network request. DCMI also publishes Qualified Dublin Core XML Schemas; this project-scoped file preserves the DCMI namespace but does not claim to reproduce the complete official schema.
